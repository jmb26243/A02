# How to create an Account and Repository on GitHub using Git and Webstorm
---

# 1 - Install Software 
- Install Webstorm via JetBrains Toolbox or directly for windows/mac/linux - https://www.jetbrains.com/webstorm/download/?section=windows 
- Install Git for the required OS - https://git-scm.com/install/windows 
---

# 2 - Create GitHub Account 
- Search for GitHub on browser
- Click the link - https://github.com/ 
- Select "Sign Up"
- Choose login method (e.g., Google, Apple, manual, etc.)
- Enter login information
- Complete security puzzle
- After completing 1-6 properly, user should be directed to the homepage of GitHub
---

# 3 - Create Repository
- Navigate to the left side of the screen and click "New Repository"
- Adjust reposiory settings if needed
- Create the repository 
---

# Glossary:
- Branch - a separate line of development within a repository that allows you to work on changes without affecting the main codebase (it 'branches' off from the main base).
- Clone - to copy a repository from a remote source (i.e., GitHub) onto your local computer.
- Commit - a saved snapshot of changes in a repository, paired with a message describing what was changed.
- Fetch - the action of downloading updates from a remote repository without automatically merging them into your current branch.
- Git - a distributed version control system used to track and manage changes in files over time.
- Github - an online platform that hosts Git repositories and supports collaboration/version control.
- Merge - the process of combining changes from one branch into another.
- Merge Conflict - occurs when Git cannot automatically combine changes from different branches because they modify the same part of a file differently.
- Push - the action of sending your committed local changes to a remote repository.
- Pull - the action of fetching changes from a remote repository and automatically merging them into your current branch.
- Remote - a version of a repository hosted on a server (i.e., GitHub) that your local repository can connect to.
- Reposiory - a Git-tracked project folder that contains files and the complete history of their changes.
